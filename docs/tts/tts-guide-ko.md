## TTS가 체스 학습을 바꾸는 이유

주석이 달린 기보를 복습할 때, 여러분의 눈은 두 가지 일을 동시에 합니다. 보드 위의 기물을 따라가면서 동시에 해설도 읽어야 합니다. 시선은 보드와 주석 패널 사이를 오가며, 그때마다 잠시 국면을 놓치게 됩니다. 기물의 위치를 다시 찾고, 수순을 다시 따라가고, 머릿속에서 국면을 다시 구성해야 합니다.

텍스트 음성 변환 기능은 이 문제를 완전히 해결합니다.

TTS를 활성화하면, 기보를 넘기기만 해도 주석이 *음성으로 읽어* 줍니다. 눈은 보드에 집중한 채로 있을 수 있습니다. 나이트가 f3에 착지하는 것을 보면서 왜 그것이 뛰어난 전개 수인지를 음성으로 들을 수 있습니다. 폰 구조가 변하는 것을 보면서 그 뒤에 있는 전략적 아이디어를 해설이 설명해 줍니다. 보드와 말이 동시에 도착합니다 — 마치 맞은편에 코치가 앉아서 가르쳐 주는 것 같은 느낌입니다.

특히 다음과 같은 상황에서 효과적입니다:

- **오프닝 연구** — 국면이 전개되는 것을 보면서 각 수 뒤에 있는 아이디어를 귀로 들을 수 있습니다
- **기보 복습** — 자신의 주석이 달린 기보를 스텝 재생하며 교훈을 자연스럽게 흡수할 수 있습니다
- **엔드게임 연습** — 중요한 칸에 집중한 채로 해설이 안내해 줍니다
- **언어 학습과의 융합** — 프랑스어, 독일어, 스페인어, 러시아어, 일본어, 중국어, 한국어로 체스를 공부할 수 있습니다. 모든 체스 용어가 자동으로 번역됩니다. "Knight f3, check" 대신 "나이트 f3, 체크"라고 들립니다. 자신이 생각하는 언어로 체스를 배우세요.
- **접근성** — 읽는 것보다 듣는 것이 편한 분이나 책상을 떠나 학습하고 싶은 분에게 최적입니다

한번 사용해 보면, 음성 없는 주석으로 돌아가는 것은 영화를 무음으로 보는 것 같은 느낌이 될 것입니다.

## 프로바이더 선택

En Parlant~에는 다섯 가지 TTS 프로바이더가 탑재되어 있습니다. OS 기본 음성부터 프리미엄 클라우드 AI 음성까지, 그리고 그 사이에 로컬 ML 모델이 있습니다. 시작하는 데 필요한 프로바이더는 하나뿐입니다. 자신에게 맞는 것을 선택하세요.

|                        | System (OS 기본)              | KittenTTS (로컬 AI)              | Google Cloud                  | ElevenLabs                    | OpenTTS (Docker)                   |
|------------------------|-------------------------------|----------------------------------|-------------------------------|-------------------------------|------------------------------------|
| **비용**               | 무료, 무제한                   | 무료, 무제한                      | 무료 (월 100만 글자)            | 무료 (월 10,000 글자)           | 무료, 무제한                        |
| **음성 품질**           | 기본 (OS 음성)                | 매우 우수 (신경망 AI)              | 매우 우수 (WaveNet 신경망)      | 탁월 (프리미엄 AI)              | 기능적 (구세대 신경망/규칙 기반)      |
| **음성 선택**           | OS에 따라 다름                | 8개 음성 (남성 4, 여성 4)          | 언어별 남성 또는 여성            | 수십 가지 고유한 캐릭터          | 75개 이상 다양한 엔진                |
| **언어**               | OS에 따라 다름                | 영어만                            | 8개 언어, CJK 우수              | 8개 언어, CJK 우수              | 유럽 언어가 가장 좋음                |
| **인터넷 필요**         | 아니요                        | 아니요 (로컬 실행)                | 예 (Google 서버)               | 예 (ElevenLabs 서버)           | 아니요 (로컬 실행)                   |
| **API 키 필요**         | 아니요                        | 아니요                            | 예                             | 예                             | 아니요                              |
| **설정 난이도**         | 없음                          | 보통 (Python + ML 모델)           | 보통 (Cloud Console)           | 쉬움 (간단한 가입)              | 쉬움 (Docker 명령 하나)             |
| **하드웨어 요구**       | 최소                          | 최신 CPU (아래 참조)              | 최소                           | 최소                           | 최소                               |
| **추천 대상**           | 빠른 테스트 / 기본 사용        | 최고의 로컬 품질                  | 대부분의 사용자                 | 음성 품질을 중시하는 분          | 프라이버시 / 오프라인 (다양한 음성)   |

### 작동 방식

**System (OS 기본)**은 운영 체제에 내장된 음성 합성을 사용합니다. Linux에서는 일반적으로 eSpeak 또는 speech-dispatcher, macOS에서는 시스템 음성, Windows에서는 SAPI입니다. 설치할 것도 없고, API 키도 없고, 서버를 실행할 필요도 없습니다. 선택하기만 하면 됩니다. 음성 품질은 기본적이지만, TTS가 작동하는지 빠르게 테스트하거나 설정 없이 기능적인 내레이션을 원할 때 완벽합니다.

**KittenTTS (로컬 AI)**는 사용자의 컴퓨터에서 완전히 실행되는 경량 신경망 텍스트 음성 변환 모델입니다. HuggingFace에서 ~25MB "nano" 모델을 사용하며 8개의 표현력 있는 음성(남성 4, 여성 4)을 제공합니다. 로컬 모델치고는 놀라울 정도로 품질이 좋습니다 — 자연스러운 억양, 선명한 발음, 진정한 표현력. 대신 하드웨어가 필요합니다: KittenTTS는 PyTorch를 사용하여 추론하며 음성 생성 중에 상당한 CPU 리소스를 소비합니다. 최신 멀티코어 CPU에서는 훌륭합니다. 구형 노트북에서는 지연이나 높은 CPU 사용량을 느낄 수 있습니다. 현재 영어만 지원합니다. **아래의 하드웨어 요구 사항을 참조하세요.**

**Google Cloud TTS**는 텍스트를 Google 서버로 전송하고, WaveNet 신경망을 사용하여 자연스러운 음성을 생성합니다. Google의 무료 한도는 넉넉합니다 — 월 100만 글자로 수백 개의 주석이 달린 기보를 커버합니다. Google Cloud 계정과 API 키가 필요하지만, 무료 한도를 초과하지 않는 한 요금이 부과되지 않습니다.

**ElevenLabs**는 프리미엄 AI 음성 플랫폼입니다. 텍스트가 서버로 전송되고 표현력 있고 사람 같은 억양의 고품질 오디오로 돌아옵니다. 음성에는 진짜 개성이 있습니다. 무료 한도는 작지만 (10,000 글자 — 기보 2~5건), 유료 플랜은 저렴합니다 (월 $5에 30,000 글자). 설정은 간단합니다: 가입하고 API 키를 복사하면 됩니다.

**OpenTTS**는 Docker를 사용하여 사용자의 컴퓨터에서 실행되는 오픈소스 서버입니다. 아무것도 컴퓨터 밖으로 나가지 않습니다. 여러 TTS 엔진(Larynx, Festival, eSpeak, Coqui-TTS 등)을 번들하여 영어만으로도 75개 이상의 음성을 제공합니다. 음성 품질은 구세대 신경망 및 규칙 기반 엔진이므로 Google이나 ElevenLabs보다 기계적입니다. CJK 문자는 잘 지원되지 않습니다. 하지만 무료, 무제한, 오프라인 TTS를 원한다면 OpenTTS가 제공합니다.

**추천:** 품질, 언어 지원, 사용 편의성의 최적 균형을 원한다면 **Google Cloud**부터 시작하세요. 무료 한도로 월 수백 개의 기보를 커버합니다. 클라우드 의존 없이 고품질 로컬 TTS를 원하고 최신 멀티코어 CPU가 있다면 **KittenTTS**가 탁월합니다. 최고의 음성 품질을 원한다면 **ElevenLabs**를 사용해 보세요. 설정 없이 바로 테스트하려면 **System TTS**가 즉시 작동합니다. 최대 프라이버시와 다양한 음성 옵션을 원한다면 **OpenTTS**가 Docker를 통해 모든 것을 로컬에서 실행합니다.

## Google Cloud TTS 설정

이 과정은 약 5분 정도 걸립니다. Google 계정(Gmail이나 YouTube에서 사용하는 것)이 필요합니다.

### 1단계: Google Cloud Console에 로그인

1. 브라우저를 열고 **[console.cloud.google.com](https://console.cloud.google.com/)** 으로 이동합니다
2. Google 계정으로 로그인합니다
3. 처음인 경우 서비스 약관에 동의해야 합니다. 체크박스를 선택하고 **Agree and Continue**를 클릭합니다

Google Cloud Console 대시보드가 표시됩니다. 화면이 복잡해 보이지만, 여기서 필요한 것은 두 가지뿐입니다.

### 2단계: 결제 설정

Google은 무료 한도라도 결제 계정을 요구합니다. **월 100만 글자를 초과하지 않는 한 요금이 부과되지 않습니다** (체스 주석으로 그 정도를 사용하기는 매우 어렵습니다). 한도에 가까워지면 Google이 미리 경고를 표시합니다.

1. 상단 검색창에 **"Billing"**을 입력하고 드롭다운에서 **Billing**을 클릭합니다
2. **Link a billing account**를 클릭합니다 (처음이면 **Create account**)
3. 안내에 따라 신용카드 또는 직불카드를 등록합니다
4. 완료되면 결제 계정 옆에 녹색 체크 표시가 나타납니다

> **참고:** 다른 프로젝트에서 이미 Google Cloud 결제를 설정한 경우 이 단계를 건너뛸 수 있습니다. 기존 결제 계정을 그대로 사용할 수 있습니다.

### 3단계: Text-to-Speech API 활성화

사용할 서비스를 Google에 알려주는 단계입니다.

1. 상단 검색창에 **"Text-to-Speech"**를 입력합니다
2. 드롭다운 결과에서 **Cloud Text-to-Speech API** (파란색 API 아이콘이 표시)를 클릭합니다
3. API 세부 정보 페이지가 열립니다. 큰 파란색 **Enable** 버튼을 클릭합니다
4. 몇 초 기다립니다. 버튼이 **Manage**로 바뀌면 API가 활성화된 것입니다

### 4단계: API 키 생성

API 키는 En Parlant~가 Google 서버와 통신하기 위해 사용하는 것입니다.

1. 상단 검색창에 **"Credentials"**를 입력하고 "APIs & Services" 아래의 **Credentials**를 클릭합니다
2. 페이지 상단 근처의 **+ Create Credentials**를 클릭합니다
3. 드롭다운에서 **API key**를 선택합니다
4. 다이얼로그에 새 키가 표시됩니다. `AIzaSyC...약 35자...`와 같은 형태입니다
5. 키 옆의 **복사 아이콘**을 클릭하여 클립보드에 복사합니다
6. **Close**를 클릭합니다

> **권장: 키를 제한하세요.** 키를 생성하면 Credentials 페이지에 목록으로 표시됩니다. 키 이름을 클릭하여 설정을 엽니다. **API restrictions** 항목에서 **Restrict key**를 선택한 다음 드롭다운에서 **Cloud Text-to-Speech API**를 선택하고 **Save**를 클릭합니다. 이렇게 하면 키가 유출되더라도 TTS 이외의 용도로는 사용할 수 없습니다.

### 5단계: En Parlant~ 설정

거의 다 되었습니다!

1. En Parlant~를 열고 **Settings** (톱니바퀴 아이콘) > **Sound** 탭으로 이동합니다
2. TTS 섹션까지 스크롤합니다
3. **TTS Provider**를 **Google Cloud**로 설정합니다
4. **Google Cloud API Key** 필드를 클릭하고 키를 붙여넣습니다 (Ctrl+V)
5. **Text-to-Speech**를 **On**으로 설정합니다
6. 음성 선택 옆의 **Test** 버튼을 클릭합니다

체스 수가 음성으로 읽어지면 설정 완료입니다!

> **문제 해결:** 테스트 음성이 나오지 않으면 다음을 확인하세요. (1) API 키가 완전히 붙여넣어졌는지, (2) Text-to-Speech API가 활성화되었는지 (3단계), (3) 결제가 연결되었는지 (2단계). 가장 흔한 원인은 API 활성화를 잊은 것입니다.

## ElevenLabs 설정

ElevenLabs는 설정이 더 간단하지만 무료 한도가 작습니다 (월 10,000 글자 — 몇 게임을 시도해 보기에는 충분합니다).

### 1단계: 계정 생성

1. 브라우저를 열고 **[elevenlabs.io](https://elevenlabs.io/)** 로 이동합니다
2. 오른쪽 상단의 **Sign Up**을 클릭합니다
3. Google, GitHub 또는 이메일로 가입할 수 있습니다. 가장 편한 방법을 선택하세요
4. 가입이 완료되면 ElevenLabs 대시보드가 표시됩니다

### 2단계: API 키 받기

1. 대시보드 왼쪽 하단의 **프로필 아이콘** (또는 이름)을 클릭합니다
2. **Profile + API key**를 클릭합니다
3. API 키 섹션이 표시됩니다. **Reveal**을 클릭하여 키를 표시하거나, 아직 키가 없으면 **Generate**를 클릭합니다
4. 키는 `sk_...약 30자...`와 같은 형태입니다
5. **복사 아이콘**을 클릭하여 클립보드에 복사합니다

### 3단계: En Parlant~ 설정

1. En Parlant~를 열고 **Settings** (톱니바퀴 아이콘) > **Sound** 탭으로 이동합니다
2. TTS 섹션까지 스크롤합니다
3. **TTS Provider**를 **ElevenLabs**로 설정합니다
4. **ElevenLabs API Key** 필드를 클릭하고 키를 붙여넣습니다 (Ctrl+V)
5. **TTS Voice** 드롭다운에 사용 가능한 음성이 표시됩니다. **Adam**은 훌륭한 기본 선택입니다 — 명확하고 자연스러우며 체스 해설에 잘 어울립니다
6. **Text-to-Speech**를 **On**으로 설정합니다
7. 음성 선택 옆의 **Test** 버튼을 클릭합니다

체스 수가 음성으로 읽어져야 합니다.

> **무료 한도에 대해:** ElevenLabs 무료 플랜에서는 월 10,000 글자까지 사용할 수 있습니다. 주석이 달린 기보 1건당 약 2,000~4,000 글자를 사용하므로 매월 2~5개의 기보를 무료로 복습할 수 있습니다. TTS가 유용하다고 느끼시면 Starter 플랜 (월 $5, 30,000 글자)을 추천합니다. Pro 플랜 (월 $22, 100,000 글자)은 헤비 사용자를 위한 것입니다.

## OpenTTS (셀프 호스팅) 설정

OpenTTS는 Docker를 사용하여 사용자의 컴퓨터에서 실행됩니다. 계정도 없고, API 키도 없고, 데이터가 컴퓨터 밖으로 나가지 않습니다. Docker가 이미 설치되어 있으면 약 2분이면 설정이 완료됩니다.

### 1단계: Docker 설치

Docker가 아직 없다면:

- **Linux:** `sudo apt install docker.io` (Ubuntu/Debian) 또는 `sudo dnf install docker` (Fedora)
- **macOS/Windows:** [Docker Desktop](https://www.docker.com/products/docker-desktop/) 다운로드

### 2단계: OpenTTS 서버 시작

터미널을 열고 실행합니다:

```bash
docker run -d -p 5500:5500 --name opentts synesthesiam/opentts:en
```

영어 음성 팩(첫 실행 시 ~1.5GB)을 다운로드하고 백그라운드에서 서버를 시작합니다. 중지할 때까지 계속 실행됩니다.

다른 언어 팩도 사용 가능합니다: `de`, `fr`, `es`, `ru`, `nl`, `sv`, `it` 등. 모든 언어를 원하면 `synesthesiam/opentts:all`을 사용하세요 (더 큰 다운로드). 예:

```bash
docker run -d -p 5500:5500 --name opentts synesthesiam/opentts:all
```

### 3단계: En Parlant~ 설정

1. En Parlant~를 열고 **Settings** (톱니바퀴 아이콘) > **Sound** 탭으로 이동합니다
2. TTS 섹션까지 스크롤합니다
3. **TTS Provider**를 **OpenTTS (Self-Hosted)**로 설정합니다
4. **OpenTTS Server URL**이 `http://localhost:5500`인지 확인합니다
5. **TTS Voice** 드롭다운에 서버에서 사용 가능한 음성이 표시됩니다. 최상의 품질을 위해 **larynx** 음성(예: `harvard`)을 사용해 보세요. 75개 이상의 옵션을 탐색할 수 있습니다
6. **Text-to-Speech**를 **On**으로 설정합니다
7. 음성 선택 옆의 **Test** 버튼을 클릭합니다

체스 수가 음성으로 읽어져야 합니다.

> **음성 품질 가이드:** OpenTTS에는 여러 엔진이 번들되어 있습니다. 최상에서 기본 순서: **Larynx** (신경망, 가장 자연스러움), **Coqui-TTS** (신경망, 다중 화자), **MaryTTS** (Java 기반, 적절), **Festival** (전통적), **eSpeak** (기계적이지만 빠름). 음성 드롭다운에 엔진 이름이 괄호 안에 표시됩니다.

> **서버 관리:** OpenTTS 컨테이너는 백그라운드에서 실행됩니다. 중지: `docker stop opentts`. 재시작: `docker start opentts`. 완전히 제거: `docker rm -f opentts`.

> **CJK 언어 참고:** OpenTTS는 유럽 언어에 최적화되어 있습니다. 일본어, 중국어, 한국어 텍스트는 영어 학습 음성으로는 올바르게 발음되지 않습니다. CJK 언어에는 Google Cloud 또는 ElevenLabs를 사용하세요.

## KittenTTS (로컬 AI) 설정

KittenTTS는 사용자의 컴퓨터에서 직접 신경망 TTS 모델을 실행합니다. 클라우드도 없고, API 키도 없고, 데이터가 컴퓨터 밖으로 나가지 않습니다 — 그리고 음성 품질이 정말 좋습니다. 설정은 약 5분 정도 걸립니다.

### 하드웨어 요구 사항

KittenTTS는 PyTorch를 사용하여 CPU에서 신경망 추론을 수행합니다. 진짜 컴퓨팅 파워가 필요합니다:

| 하드웨어 | 경험 |
|----------|------|
| **8코어 이상, 최신 CPU** (Ryzen 7, i7, Xeon) | 탁월함. 음성 생성이 빠르고 다른 작업에 미치는 영향이 최소 |
| **4-6코어** (Ryzen 5, i5) | 좋음. 생성 중 눈에 띄는 CPU 사용량이지만 충분히 사용 가능 |
| **2코어 / 구형 CPU** | 느림. 발화당 수 초가 걸릴 수 있음. Google Cloud를 고려하세요 |

**CPU 사용량은 일시적입니다** — KittenTTS는 음성을 생성하는 동안(일반적으로 발화당 1-2초)에만 CPU를 사용합니다. 발화 사이에 CPU 사용량은 거의 0으로 떨어집니다.

**스레드 관리:** 기본적으로 KittenTTS는 최대 생성 속도를 위해 사용 가능한 모든 CPU 코어를 사용합니다. 체스 엔진(예: Stockfish)도 함께 실행하고 있다면 KittenTTS가 사용하는 스레드 수를 제한할 수 있습니다. **Settings > Sound > KittenTTS CPU Threads**에서 스레드 사용량을 제한하세요. 0으로 설정하면 자동(모든 코어 사용)입니다. 체스 엔진과 공유 사용 시 좋은 시작점은 코어 수의 절반입니다.

> **중요:** KittenTTS와 Stockfish 같은 체스 엔진 모두 CPU 코어를 원합니다. Stockfish로 깊은 분석을 하면서 KittenTTS가 음성을 생성하면 CPU 시간을 놓고 경쟁합니다. 스레드 설정으로 이를 균형잡을 수 있습니다. 8코어 이상이면 거의 느끼지 못합니다. 4코어에서는 KittenTTS에 2개 스레드를 주고 나머지는 엔진에 남겨두는 것이 좋습니다.

### 1단계: 의존성 설치

KittenTTS에는 Python 3.10+ 이상과 몇 가지 Python 패키지가 필요합니다. 두 가지 방법으로 설정할 수 있습니다:

**방법 A: 앱 내 설정 마법사 (권장)**

1. En Parlant~를 열고 **Settings > Sound**로 이동합니다
2. **TTS Provider**를 **KittenTTS (English Only)**로 설정합니다
3. 의존성이 없으면 노란색 "Setup Guide" 알림이 나타납니다
4. 알림을 클릭하면 설정 마법사가 열립니다
5. 마법사가 각 단계를 안내하며 자동 설치를 위한 "Fix" 버튼을 제공합니다

**방법 B: 터미널 설정 스크립트**

```bash
cd /path/to/en-parlant
./scripts/setup-tts.sh --kittentts
```

Python 가상 환경을 생성하고 필요한 패키지(kittentts, flask, soundfile, numpy)를 설치합니다. nano 모델(~25MB)은 첫 실행 시 HuggingFace에서 다운로드됩니다.

**방법 C: 수동 설정**

```bash
cd /path/to/en-parlant/scripts
python3 -m venv .venv
.venv/bin/pip install kittentts flask soundfile numpy
```

### 2단계: En Parlant~ 설정

1. En Parlant~를 열고 **Settings** (톱니바퀴 아이콘) > **Sound** 탭으로 이동합니다
2. **TTS Provider**를 **KittenTTS (English Only)**로 설정합니다
3. 프로바이더를 선택하면 서버가 자동으로 시작됩니다
4. 모델이 로드될 때까지 몇 초 기다립니다 (첫 실행 시 HuggingFace에서 다운로드)
5. 음성을 선택합니다 — 8가지 옵션(남성 4, 여성 4)이 있습니다
6. 음성 선택 옆의 **Test** 버튼을 클릭합니다

자연스럽고 표현력 있는 AI 음성으로 체스 수가 읽어져야 합니다.

> **첫 실행:** KittenTTS를 처음 사용할 때 ~25MB nano 모델이 HuggingFace에서 다운로드됩니다. 일회성 다운로드입니다. 이후 시작은 더 빠릅니다 (일반적으로 모델을 메모리에 로드하는 데 2-5초).

> **음성 가이드:** KittenTTS는 2-5번까지 번호가 매겨진 8개의 음성을 제공하며 각각 남성과 여성 변형이 있습니다. 모든 음성은 영어이며 약간씩 다른 톤 특성을 가지고 있습니다. 몇 가지를 시도해 보고 마음에 드는 것을 찾으세요.

> **영어만 지원:** KittenTTS는 현재 영어만 지원합니다. 다른 언어에는 Google Cloud 또는 ElevenLabs를 사용하세요. KittenTTS 사용 시 TTS 언어 설정은 무시됩니다 — 체스 용어는 항상 영어로 말합니다.

## System TTS (OS 기본) 설정

System TTS는 운영 체제에 내장된 음성 합성을 사용합니다. 설치할 것이 없습니다.

### En Parlant~ 설정

1. En Parlant~를 열고 **Settings** (톱니바퀴 아이콘) > **Sound** 탭으로 이동합니다
2. **TTS Provider**를 **System (OS Native)**으로 설정합니다
3. 음성 드롭다운에 시스템에서 사용 가능한 모든 음성이 표시됩니다
4. **Test** 버튼을 클릭하여 미리 듣기합니다

그게 전부입니다. System TTS는 설정 없이 즉시 작동합니다.

> **음성 품질:** 시스템 음성은 OS마다 다릅니다. macOS 음성이 가장 자연스러운 경향이 있습니다. Linux 음성(일반적으로 eSpeak 또는 speech-dispatcher를 통한 Festival)은 더 기계적입니다. Windows SAPI 음성은 그 중간입니다. 품질이 너무 기본적이라면 KittenTTS(로컬) 또는 Google Cloud로 업그레이드하세요.

> **언어 지원:** 사용 가능한 언어는 OS와 설치된 음성 팩에 따라 다릅니다. Linux에서는 추가 음성을 설치할 수 있습니다: `sudo apt install espeak-ng-data-*`

## 지원 언어

TTS 내레이션은 현재 체스 용어가 완전히 번역된 8개 언어를 지원합니다:

| 언어                | 체스 예시                                             |
|--------------------|------------------------------------------------------|
| **English**        | Knight f3, check. A strong developing move.          |
| **Français**       | Cavalier f3, échec. Un coup de développement fort.   |
| **Español**        | Caballo f3, jaque. Un fuerte movimiento.             |
| **Deutsch**        | Springer f3, Schach. Ein starker Entwicklungszug.    |
| **日本語**          | ナイト f3、チェック。強い展開の手。                       |
| **Русский**        | Конь f3, шах. Сильный развивающий ход.               |
| **中文**            | 马 f3，将军。一步控制中心的强力出子。                     |
| **한국어**          | 나이트 f3, 체크. 중앙을 지배하는 강력한 전개 수.          |

기물 이름, "체크", "체크메이트", "캐슬링", "잡음", 그리고 "브릴리언트한 수"와 "대실수" 같은 수의 평가 — 모든 체스 용어가 선택한 언어로 읽어집니다. PGN 파일의 코멘트는 그대로 읽어지므로, 듣고 싶은 언어로 기보에 주석을 달아 주세요.

## 더 나은 경험을 위한 팁

- **Auto-Narrate를 활용하세요.** "Auto-Narrate on Move"를 켜고 화살표 키로 기보를 넘기기만 하면 됩니다. 수를 넘길 때마다 해설이 자연스럽게 흘러나옵니다. 마치 어깨 너머로 코치가 있는 것 같은 느낌입니다.

- **자신의 대국에 주석을 달아 보세요.** TTS가 진가를 발휘하는 것은 *자신의* 대국 해설을 들을 때입니다. 대국에 주석을 단 후 내레이션과 함께 스텝 재생해 보세요. 국면을 바라보면서 "이 폰을 잡는 것은 매력적으로 보이지만, 킹사이드는 아직 전혀 전개되지 않았습니다"라고 들려오는 경험은 텍스트로 읽는 것과는 전혀 다릅니다.

- **재생 속도를 바꿔 보세요.** 꼼꼼히 연구하려면 1배속, 빠르게 복습하려면 1.3배속을 선호하는 분이 많습니다. 속도 슬라이더는 실시간으로 재생 속도를 조절하며 추가 API 글자 수를 소비하지 않습니다. 오디오는 한 번만 생성되고 재생 속도만 바뀝니다.

- **스피커 아이콘을 활용하세요.** 수순 목록의 각 코멘트에는 작은 스피커 아이콘이 있습니다. 클릭하면 전체 기보를 넘기지 않고 해당 주석만 들을 수 있습니다.

- **언어를 바꿔서 체스 용어를 배우세요.** 제2언어로 체스를 공부하고 있다면 TTS 언어를 맞춰 보세요. 듣기만 해도 "Cavalier" (나이트), "échec" (체크), "mat" (체크메이트) 같은 용어가 자연스럽게 익혀집니다.
